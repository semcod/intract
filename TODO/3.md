Tak — Intract najlepiej wdrożyć w reDUP jako **osobny detektor duplikacji intencji**, działający obok obecnych detektorów `exact`, `structural`, `near`, `fuzzy` i `semantic`.

## Plik

Kod koncepcyjny:

```python id="o0i807"
from __future__ import annotations

from pathlib import Path
from typing import Iterable

from redup.core.scanner_types import CodeBlock


def block_text(block: CodeBlock) -> str:
    return (
        getattr(block, "content", "")
        or getattr(block, "text", "")
        or getattr(block, "source", "")
        or ""
    )


def block_file_path(block: CodeBlock) -> str:
    return str(
        getattr(block, "file_path", "")
        or getattr(block, "path", "")
        or ""
    )


def block_start_line(block: CodeBlock) -> int:
    return int(
        getattr(block, "start_line", 1)
        or getattr(block, "start", 1)
        or 1
    )


def block_end_line(block: CodeBlock) -> int:
    return int(
        getattr(block, "end_line", block_start_line(block))
        or getattr(block, "end", block_start_line(block))
        or block_start_line(block)
    )


def signatures_from_blocks(blocks: Iterable[CodeBlock]):
    try:
        from intract.parser import extract_contract_records_from_text
        from intract.signature import build_signature
    except ImportError as exc:
        raise RuntimeError(
            "Intract support requires installing redup[intract] or intract>=0.1.0"
        ) from exc

    signatures = []
    blocks_by_signature_id = {}

    for block in blocks:
        text = block_text(block)
        file_path = block_file_path(block)

        if "@intract" not in text and "@ridl" not in text:
            continue

        records = extract_contract_records_from_text(
            text,
            file_path=file_path,
            default_scope="function",
        )

        for record in records:
            signature = build_signature(record)
            signatures.append(signature)
            blocks_by_signature_id[signature.block_id] = block

    return signatures, blocks_by_signature_id
```

To daje minimalny MVP: Intract działa tylko na blokach, które mają `@intract.v1`.

Kod koncepcyjny:

```python id="41h7yg"
from __future__ import annotations

from collections import defaultdict
from dataclasses import dataclass
from itertools import combinations


@dataclass(frozen=True)
class IntentPair:
    left_id: str
    right_id: str
    score: float
    reason: dict[str, object]


def jaccard(a: set[str], b: set[str]) -> float:
    if not a and not b:
        return 1.0
    if not a or not b:
        return 0.0
    return len(a & b) / len(a | b)


def object_similarity(left: str, right: str) -> float:
    if left == right:
        return 1.0

    return jaccard(set(left.split("_")), set(right.split("_")))


def score_contract_similarity(left, right) -> tuple[float, dict[str, object]]:
    action_score = 1.0 if left.action == right.action else 0.0
    object_score = object_similarity(left.object, right.object)

    domain_score = 1.0
    if left.domain or right.domain:
        domain_score = 1.0 if left.domain == right.domain else 0.0

    input_score = jaccard(set(left.inputs), set(right.inputs))
    output_score = jaccard(set(left.outputs), set(right.outputs))
    effect_score = jaccard(set(left.effects), set(right.effects))
    tag_score = jaccard(set(left.tags), set(right.tags))

    score = (
        0.28 * action_score
        + 0.26 * object_score
        + 0.14 * domain_score
        + 0.10 * input_score
        + 0.10 * output_score
        + 0.06 * effect_score
        + 0.06 * tag_score
    )

    return round(score, 4), {
        "action_score": action_score,
        "object_score": object_score,
        "domain_score": domain_score,
        "input_score": input_score,
        "output_score": output_score,
        "effect_score": effect_score,
        "tag_score": tag_score,
        "left": left.key,
        "right": right.key,
    }


def bucket_signatures(signatures):
    buckets = defaultdict(list)

    for signature in signatures:
        # Pierwszy tani bucketing: po akcji.
        buckets[signature.action].append(signature)

    return buckets


def find_intent_pairs(signatures, threshold: float = 0.84) -> list[IntentPair]:
    pairs = []

    for bucket in bucket_signatures(signatures).values():
        for left, right in combinations(bucket, 2):
            if left.block_id == right.block_id:
                continue

            if left.exact_hash == right.exact_hash:
                pairs.append(
                    IntentPair(
                        left_id=left.block_id,
                        right_id=right.block_id,
                        score=1.0,
                        reason={"exact_contract_hash": True},
                    )
                )
                continue

            score, reason = score_contract_similarity(left, right)

            if score >= threshold:
                pairs.append(
                    IntentPair(
                        left_id=left.block_id,
                        right_id=right.block_id,
                        score=score,
                        reason=reason,
                    )
                )

    return pairs
```

# 9. Grupowanie par w DuplicateGroup

## Plik

```text id="qct95k"
src/redup/integrations/intract/grouping.py
```

```python id="7me1yv"
from __future__ import annotations

from collections import defaultdict

from redup.core.models import DuplicateFragment, DuplicateGroup, DuplicateType


def union_find_groups(pairs):
    parent = {}

    def find(node):
        parent.setdefault(node, node)
        if parent[node] != node:
            parent[node] = find(parent[node])
        return parent[node]

    def union(left, right):
        root_left = find(left)
        root_right = find(right)
        if root_left != root_right:
            parent[root_right] = root_left

    for pair in pairs:
        union(pair.left_id, pair.right_id)

    groups = defaultdict(set)

    for node in parent:
        groups[find(node)].add(node)

    return [group for group in groups.values() if len(group) > 1]


def pairs_to_duplicate_groups(pairs, signatures_by_id, blocks_by_signature_id):
    grouped_ids = union_find_groups(pairs)
    duplicate_groups = []

    pair_scores = {
        frozenset([pair.left_id, pair.right_id]): pair
        for pair in pairs
    }

    for index, ids in enumerate(grouped_ids, start=1):
        fragments = []

        for signature_id in ids:
            signature = signatures_by_id[signature_id]
            block = blocks_by_signature_id.get(signature_id)

            fragments.append(
                DuplicateFragment(
                    file_path=signature.file_path,
                    start_line=signature.start_line,
                    end_line=signature.end_line,
                    content=getattr(block, "content", "") if block else "",
                    normalized_hash=signature.exact_hash,
                )
            )

        scores = [
            pair.score
            for key, pair in pair_scores.items()
            if key.issubset(ids)
        ]

        avg_score = sum(scores) / max(1, len(scores))

        duplicate_groups.append(
            DuplicateGroup(
                group_id=f"intent_{index:04d}",
                duplicate_type=DuplicateType.INTENT,
                similarity=round(avg_score, 4),
                fragments=fragments,
                normalized_hash="intent:" + str(index),
                metadata={
                    "engine": "intract",
                    "contract_ids": sorted(ids),
                    "scores": scores,
                },
            )
        )

    return duplicate_groups
```

Uwaga: nazwy pól `DuplicateFragment` / `DuplicateGroup` trzeba dopasować do faktycznej definicji w `models.py`, ale kierunek jest taki.

# 10. Detektor Intract dla reDUP

## Plik

```text id="cxad15"
src/redup/integrations/intract/detector.py
```

```python id="8uy140"
from __future__ import annotations

from pathlib import Path

from redup.integrations.intract.adapter import (
    signatures_from_blocks,
    signatures_from_manifest,
)
from redup.integrations.intract.matcher import find_intent_pairs
from redup.integrations.intract.grouping import pairs_to_duplicate_groups


def find_intract_intent_groups(blocks, config):
    if not getattr(config, "intent_enabled", False):
        return []

    threshold = getattr(config, "intent_threshold", 0.84)
    manifest = getattr(config, "intent_manifest", None)

    signatures, blocks_by_signature_id = signatures_from_blocks(blocks)

    if manifest:
        manifest_signatures = signatures_from_manifest(manifest)
        signatures.extend(manifest_signatures)

    if not signatures:
        return []

    signatures_by_id = {signature.block_id: signature for signature in signatures}

    pairs = find_intent_pairs(
        signatures,
        threshold=threshold,
    )

    if not pairs:
        return []

    return pairs_to_duplicate_groups(
        pairs,
        signatures_by_id=signatures_by_id,
        blocks_by_signature_id=blocks_by_signature_id,
    )
```

```text id="brfd3j"
  fuzzy duplicate detector
  intract intent duplicate detector
  semantic duplicate detector
```

Intract nie zastępuje reDUP. Intract dodaje nowy wymiar:

```text id="f1bt82"
duplikacja kodu        → reDUP
duplikacja intencji    → Intract inside reDUP
walidacja intencji     → Intract / vallm
```

Najważniejsze: **w reDUP używaj Intract tylko do wyszukiwania duplikacji intencji**, a walidację poprawności implementacji zostaw jako osobną funkcję Intract/vallm. Dzięki temu reDUP nie rozmyje swojej roli i nadal pozostanie analizatorem duplikacji/refaktoryzacji.
