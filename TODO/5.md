Dodałem do paczki **Intract** warstwę pluginów, SDK-template’y dla różnych języków i gotowe konfiguracje do łatwego podpięcia w nowym projekcie.

Co dodałem:

```text
src/intract/plugins/
  __init__.py
  base.py
  manager.py
  builtins.py

src/intract/sdk.py

sdks/
  python/
  typescript/
  go/
  rust/
  java/
  csharp/

templates/
  intract.yaml
  pyproject-intract.toml
  .pre-commit-config.yaml
  .github/workflows/intract.yml
  openapi.intract.yaml
  Dockerfile.intract

docs/
  plugins.md
  quick-enable.md
```

Działa już podstawowy plugin registry:

```text
ParserPlugin
ValidatorPlugin
ReporterPlugin
IntegrationPlugin
PluginRegistry
```

Dodałem też entry points w `pyproject.toml`, żeby zewnętrzne paczki mogły rejestrować pluginy:

```toml
[project.entry-points."intract.parsers"]
inline = "intract.plugins.builtins:InlineContractParserPlugin"
manifest = "intract.plugins.builtins:ManifestParserPlugin"

[project.entry-points."intract.validators"]
basic = "intract.plugins.builtins:BasicContractValidatorPlugin"

[project.entry-points."intract.reporters"]
json = "intract.plugins.builtins:JsonReporterPlugin"
```

Do nowego projektu można teraz wejść jednym z wariantów:

```bash
intract init .
intract validate . --manifest intract.yaml
```

albo przez konfigurację:

```toml
[tool.intract]
manifest = "intract.yaml"
mode = "changed"
fail_on = ["violation", "invalid_manifest", "missing_required_p1"]
```

Dodałem też SDK helpery do generowania kontraktu jako jednej linijki, np. TypeScript:

```ts
inlineContract({
  scope: "function",
  intent: "validate:user_permission",
  priority: 1,
  domain: "security",
  input: ["user", "resource"],
  output: ["allowed"],
  forbid: ["network", "write"],
  validate: ["input_presence", "return_value", "no_forbidden_effect"]
})
```

oraz analogiczne szkielety dla Go, Rust, Java i C#.

Szybki smoke test przeszedł: pluginy znalazły 1 kontrakt i uruchomiły validator `basic`.
